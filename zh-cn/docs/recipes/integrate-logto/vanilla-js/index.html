原生 JS：集成 @logto/browser

提示

Browser SDK 是 框架无关 的，也就是说只要你写一层适配框架的 SDK 包装，即可在任意一种前端框架中使用。这也是 @logto/react 和 @logto/vue 做的事。

备注

本篇教程默认你已经在管理界面中成功创建了一个「单页应用」类型的应用。如果你还没有完成这一步操作，请参阅这篇教程，创建应用之后再继续。

将 Logto SDK 添加至依赖​

  • npm
  • Yarn
  • pnpm
npm i @logto/browser
yarn add @logto/browser
pnpm add @logto/browser

初始化 LogtoClient​

引入并通过配置初始化 LogtoClient：

import LogtoClient from '@logto/browser';

const logtoClient = new LogtoClient({
  endpoint: '<your-logto-endpoint>', // 例如 http://localhost:3001
  appId: '<your-application-id>',
});
备注

在接下来的代码片段中，我们假设你的应用运行在 http://localhost:1234。

登录​

登录流程可以被简化为：

配置 Redirect URI​

让我们在这里切换到「管理控制台」的应用详情页。添加一个值为 http://localhost:1234/callback 的 Redirect URI 并点按「保存变更」。

Redirect URI 是一个 OAuth 2.0 的概念，用于表明在 authentication 后重定向至的位置。

实现登录按钮​

备注

在调用 .signIn() 之前，请先确保已经在「管理控制台」中正确配置了 Redirect URI。

返回你的 IDE 或编辑器，使用如下代码来实现一个登录按钮：

<button onclick="logtoClient.signIn('http://localhost:1234/callback')">登录</button>

处理重定向​

马上就要大功告成！在上一步，我们将 http://localhost:1234/callback 用作 Redirect URI，现在我们需要对其妥善处理。在你的 /callback 路由下插入如下代码：

try {
  await logtoClient.handleSignInCallback();
  console.log(logtoClient.isAuthenticated); // true
} catch {
  // 处理错误
}

现在可以测试登录流程了。

退出登录​

调用 .signOut() 将清理内存与 localStorage 中的所有 Logto 数据（如果有）。

在退出登录后，让你的用户重新回到你的网站是个不错的选择。让我们将 http://localhost:1234 添加至「管理控制台」里的 Post Sign-out URIs 中（位于 Redirect URIs 下方），并将其作为调用 .signOut() 的参数。

实现退出登录按钮​

<button onclick="logtoClient.signOut('http://localhost:1234')">退出登录</button>
