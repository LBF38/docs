📱 应用

简介​

应用 是用于标识你接入 Logto 中的应用程序，你将通过它来使用 OIDC 服务、配置登录体验、查看审计日志等。

如果你想要在你的应用程序中集成 Logto，详见在你的应用中集成 Logto。

属性​

应用 ID​

应用 ID 是一个自动生成的用来在 Logto 中标识你应用的唯一 key，在 OAuth 2.0 中用作 client id。

应用类型​

应用 应为以下的几种类型之一：

  • 原生应用 是运行在原生环境中的应用。例如 iOS 应用、Android 应用。
  • 单页应用 是运行在 web 浏览器中的应用，并且该应用无需重新载入整个页面，即可从服务器更新页面中的数据。例如 React DOM 应用、Vue 应用。
  • 传统 web 应用 是由 web 服务器全权负责渲染与更新页面的应用。例如 JSP、PHP。

应用名称​

应用名称 是在控制台中显示的可读的应用名称。

描述​

描述 是在控制台应用详情页中显示的关于应用的简短的描述。

Authorization Endpoint​

Authorization Endpoint 是一个 OIDC 术语，该 Endpoint 用来进行用户的身份验证。

在 Logto 中，Authorization Endpoint 将为用户唤起授权流程。

详见 Authorization Endpoint 以了解更多信息。

Redirect URIs​

允许用户在 Logto 登录后将重定向回应用程序的 URI 列表。

它将用于验证授权请求中使用的 Redirect URI，因此登录时每个要使用的 Redirect URI 都应预先添加到此允许列表中。

详见 Redirection Endpoint 以了解更多信息。

Post Sign Out Redirect URI​

允许用户在 Logto 退出登录后将重定向回应用程序的 URI 列表。

它将用于验证退出登录请求中使用的重定向 URI，因此退出登录时每个要使用的重定向 URI 都应预先添加到此允许列表中。

详见 RP-Initiated Logout 以了解更多信息。

CORS Allowed Origins​

被允许向 Logto 发起请求的 Origins。

所有需要被允许的用于用户向 Logto 服务发起请求的 Origins 都应被添加到此列表中。

通常，CORS Allowed Origins 应该包含你的应用程序所部署的地址。

Token Endpoint​

Token Endpoint 是一个 OIDC 术语, 客户端通过访问该 Enpoint 来进行授权认证以获得访问令牌，或刷新已有的访问令牌。

详见 Token Endpoint 以了解更多信息。
